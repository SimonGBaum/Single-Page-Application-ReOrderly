# ReOrderly Application
## Fundamental Questions
### What is my application?
-	A repository that allows a user to create, update, track and set delivery frequency all in one simple location. Track:
    - The date created
    - The date ordered
    - The delivery date
    - The frequency of deliveries
### What Problem does this application Solve?
This application helps the customer:
-	Remember orders
-	Remind them when an order is going to be delivered
-	To adjust the frequency of an order
-	Adjust an order to fit their needs
-	Track their current and past orders
This helps create organization and flexibility:
-	Organize their orders
-	Makes the ordering process more efficient
-	Makes ordering more flexible and efficient
This application helps resolve:
-	Trying to remember the multitude of orders that we make every day/week/month/year.
### Who Are the Users?
Who will use the application?
-	Anyone who orders medications, food stuffs, hygiene/health products: Anything that a user wants to be ordered on a recurring basis
Are there multiple user Types?
-	No
What are their responsibilities?
-	Provide personal information and product information
    - Personal Information:
        - mail, name
    - Item Information:
        - product name
        - product origin (what store did they order if from)
        - how frequently they want it to be reordered
What information does each user need?
-	Access to their profile
-	User Profile with have:
    - Frist Name
    - Last Name
    - Email
    - Mailing Address
    - Billing Address
    - Username
    - Password
### What Are the User Goals? (Clarify Your Features)
